AI와 함께 일하는 개발자

LLM 개발자로서의 철학

저는 LLM을 단순한 코드 생성 도구가 아닌, 사고 과정을 함께하는 협업자로 활용합니다. AI와의 협업을 통해 개발 생산성을 높이면서도, 품질을 담보하는 체계적인 프로세스를 추구합니다.

  • LLM은 협업자다
    자동화 대상이 아닌, 아이디어를 검증하고 확장하는 파트너입니다. 단순히 코드를 대신 작성하게 하는 것이 아니라, 함께 문제를 분석하고 해결책을 모색합니다.
  • 맥락이 곧 성능이다
    RAG든 프롬프트든, 적절한 맥락 제공이 LLM 활용의 핵심입니다. 좋은 결과를 얻기 위해서는 명확한 요구사항과 충분한 배경 정보가 필요합니다.
  • 평가 없이 개선 없다
    정량적 벤치마크와 사용자 피드백으로 지속적으로 개선합니다. 감에 의존하지 않고, 측정 가능한 지표를 통해 품질을 관리합니다.

바이브코딩 워크플로우

바이브코딩은 AI와의 협업을 통해 개발 생산성을 극대화하는 방법론입니다. 도메인 분석부터 리뷰까지, 각 단계에서 AI를 효과적으로 활용합니다.

각 단계별 설명

  • 도메인 분석: 새로운 도메인의 기술 문서를 AI로 분석하여 빠르게 이해
  • 설계: AI와의 대화를 통해 아키텍처 설계 및 API 명세 작성
  • 개발: Claude Code SKILL을 활용한 반복 작업 자동화
  • 리뷰: AI의 코드 리뷰 + 인간의 최종 검수로 품질 확보

SKILL 활용 예시

Claude Code SKILL을 생성하여 반복 작업을 자동화합니다.

# PR 메시지 생성 SKILL

## 트리거
사용자가 "PR 만들어줘", "커밋 메시지 작성" 요청 시

## 동작
1. git diff 분석
2. 변경 파일 목록 추출
3. 커밋 컨벤션에 맞는 메시지 생성

## 출력 형식
### Summary
- 변경 사항 요약 (1-3줄)

### Changes
- 파일별 변경 내용

활용 효과:

  • PR 작성 시간: 15분 → 2분
  • 컨벤션 준수율: 100%
  • 리뷰어 이해도 향상
